Proof Requirements
All resolution actions (Mark Received, Full/Partial Return, Paid Sample) REQUIRE a file attachment as proof (RGA, Photo, or PDF document). Form submission will be blocked if proof is missing.

External System Features:
• SMTP email notifications to external recipients

• Admin override capability for special cases

admin_panel_settings General Policies

verified_user

Account Approval Policy

All user accounts must be approved by an administrator before access is granted to the transmittal system. Users will remain in "Pending" status until reviewed and approved.

timer

Session Timeout Policy

For security, user sessions automatically expire after 15 minutes of inactivity. Users will be logged out and redirected to the login page. Activity such as mouse movement, clicks, or keyboard input resets the timer.

lock

Digital Signature Policy

All receipt confirmations require a digital signature from the recipient. This ensures non-repudiation and provides a verifiable record of document handover.

domain Internal Transmittal Policies

people

Custodian Responsibility

Each location must have an assigned custodian responsible for:

  • Receiving incoming transmittals at their location
  • Marking transmittals as "Arrived" upon physical receipt
  • Notifying the intended recipient for pick-up
  • Maintaining accurate records of all handovers
cancel

Cancellation Policy

Only the sender of a transmittal can cancel it, and only while it is still in "In Transit" status. Once a transmittal has been marked as Arrived, Picked, or Received, it cannot be cancelled.

notifications_active

Email Notification Policy

Automated email notifications are sent at every status change:

  • In Transit: Recipient and destination custodian are notified
  • Arrived: Recipient is notified that their package is ready for pick-up
  • Picked Up: Sender is notified
  • Received: Sender receives confirmation with digital signature
autorenew

Auto-Receive Policy

Transmittals that remain in "Picked Up" status for more than 3 days are automatically marked as "Received" by the system. This ensures transmittals are not left indefinitely in the picked-up state. An automated email notification is sent to both the sender and recipient when auto-receive occurs.

alarm

Status Reminder Policy

The system sends daily email reminders for transmittals that have been stuck in the same status for 5 or more days. This applies to transmittals in:

  • In Transit — package has not arrived at the destination
  • Arrived — package has not been picked up by the recipient
  • Picked Up — recipient has not confirmed receipt

Reminders are sent once per day to both the sender and recipient until the transmittal progresses to the next status. Received and cancelled transmittals do not trigger reminders.

public External Transmittal Policies

attach_file

Proof of Delivery Requirement

All resolution actions for external transmittals REQUIRE a file attachment as proof. Accepted formats include photos, scanned documents (PDF), or RGA forms. The system will block form submission if no proof is provided.

event

Return Deadline Policy

"For Return" transmittals must include both an Expected Return Date and a Hard Deadline. The system monitors these dates and sends automated reminders as deadlines approach. Overdue items are flagged for immediate attention.

swap_vert

Resolution Options Policy

"For Return" transmittals can be resolved through one of the following actions:

  • Full Return: All items returned — closes the transmittal
  • Partial Return: Some items returned — transmittal remains Open
  • Paid Sample: Receiver keeps the item and pays — closes the transmittal
  • Convert to Keep: Decision to make the transfer permanent — closes the transmittal
schedule_send

Deadline Reminder Policy

For "For Return" external transmittals, the system sends daily deadline reminder emails starting from the hard deadline date. Reminders are scheduled on weekdays (Monday–Friday) at 9:00 AM and continue every day until the transmittal is resolved or closed.

  • Only one reminder email is sent per transmittal per day (tracked via last notification date) to prevent duplicate emails
  • Emails are sent to the external recipient with all original file attachments included
  • Urgency escalates visually based on how overdue the transmittal is:
    • Deadline Day: "DEADLINE TODAY" (orange)
    • 1–2 Days Overdue: days overdue count (red)
    • 3–6 Days Overdue: days overdue count (dark red)
    • 7+ Days Overdue: "FINAL REMINDER" (darkest red)
  • Closed transmittals are automatically excluded from reminders

security Data & Security Policies

history

Audit Trail Policy

Every action performed in the system is recorded in the activity log, including:

  • User logins and logouts (with IP address and device info)
  • Transmittal creation, status changes, and updates
  • Administrative actions (approvals, rejections, overrides)
  • All records include timestamps for traceability
shield

Role-Based Access Control

The system enforces strict role-based permissions:

  • Custodian: Location-based management of incoming/outgoing transmittals
  • Regular User: Can send and receive transmittals within approved scope
